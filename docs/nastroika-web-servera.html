Настройка Web сервера

Настройка Web сервера

Технические характеристики сервера

Debian 10, ngnix 1.14, php 7.3

Исходные данные

Отдельный пользователь для sftp доступа и доступа по ssh: lara

Домашняя директория: /home/lara/

Папка приложения: /home/lara/www/

Доменое имя приложения: avtostars72.abp-i.online

Базовые приготовления

Устанавливаем все необходимые пакеты для работы.

apt-get install mc wget htop curl nginx mariadb-server php-fpm php-cli php-mysqlnd php-json php-gd php-ldap php-odbc php-pdo php-opcache php-pear php-xml php-xmlrpc php-mbstring php-snmp php-zip

Добавляем группу для работы по sftp протоколу

groupadd sftp

Добавляем пользователя lara

useradd -s /bin/bash lara -d /home/lara
passwd lara
usermod -a -G sftp lara

Формирование файловой структуры

mkdir -p /home/lara/{www,log,session}
chown -R lara. /home/lara

chmod -R 0775 /home/lara/
chmod 0755 /home/lara/{www,log,session}

mkdir -p /var/cache/nginx/client_temp
chown -R lara /var/cache/nginx/

mkdir -p /var/cache/nginx/proxy_temp
chown -R lara /var/cache/nginx/proxy_temp

Устанавилваем файл подкачки (если на сервере менее 1Гб RAM)

Создаем файл подкачки

dd if=/dev/zero of=/swap bs=1024 count=1000000
mkswap /swap
chmod 0600 /swap

Включаем swap

swapon /swap

Устанавилваем автоматическое монтирование файла swap.

Редактируем файл /etc/fstab добавление строки

/swap swap swap defaults 0 0

Настройка Nginx

Останавливаем службу

systemctl stop nginx

Удаляем ссылку на дефотный хост

rm /etc/nginx/sites-enabled/default

Создаем конфиг для отдельного хоста (конкретного сайта)

touch /etc/nginx/sites-available/avtostars72.abp-i.online

Правим конфиг /etc/nginx/sites-available/avtostars72.abp-i.online

server {
    listen 443 ssl http2;
    server_name avtostars72.abp-i.online;
    root /home/lara/www/public;
    index index.php;
    access_log /home/lara/log/access.log main;
    error_log /home/lara/log/error.log;

    ssl_certificate             /etc/letsencrypt/live/avtostars72.abp-i.online/fullchain.pem;
    ssl_certificate_key         /etc/letsencrypt/live/avtostars72.abp-i.online/privkey.pem;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-XSS-Protection "1; mode=block";
    add_header X-Content-Type-Options "nosniff";

    error_page 404 /index.php;

    location / {
        try_files $uri $uri/ /index.php?$args;
    }

    location ~* ^.+.(js|css|png|jpg|jpeg|gif|ico|woff)$ {
        access_log off;
        expires max;
    }

    location ~ \.php$ {
        try_files  $uri =404;
        fastcgi_pass   unix:/run/php/php7.3-fpm-lara.sock;
        fastcgi_index index.php;
        fastcgi_param DOCUMENT_ROOT /home/lara/www/public/;
        fastcgi_param SCRIPT_FILENAME /home/lara/www/public$fastcgi_script_name;
        fastcgi_param PATH_TRANSLATED /home/lara/www/public$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_param QUERY_STRING $query_string;
        fastcgi_param REQUEST_METHOD $request_method;
        fastcgi_param CONTENT_TYPE $content_type;
        fastcgi_param CONTENT_LENGTH $content_length;
        fastcgi_param HTTPS on;
        fastcgi_intercept_errors on;
        fastcgi_ignore_client_abort off;
        fastcgi_connect_timeout 60;
        fastcgi_send_timeout 180;
        fastcgi_read_timeout 180;
        fastcgi_buffer_size 128k;
        fastcgi_buffers 4 256k;
        fastcgi_busy_buffers_size 256k;
        fastcgi_temp_file_write_size 256k;
    }
    location = /favicon.ico {
        log_not_found off;
        access_log off;
    }

    location = /robots.txt {
        allow all;
        log_not_found off;
        access_log off;
    }

    location ~ /\.ht {
        deny all;
    }
}

server {

Создадим символическую ссылку для нового хоста.

ln -s /etc/nginx/sites-available/avtostars72.abp-i.online /etc/nginx/sites-enabled/avtostars72.abp-i.online

Командой проверяем что конфиг настроен верно

nginx -t

И останавливаем работу nginx пока не закончиться процесс получения сертификата

Настройка PHP

Настроим для приложения отдельный конфиг

cd /etc/php/7.3/fpm/pool.d/
cp www.conf avtostars72.abp-i.online.conf

Правик конфиг /etc/php/7.3/fpm/pool.d/avtostars72.abp-i.online.conf

[avtostars72.abp-i.online]
user = lara    
group = lara    
listen = /run/php/php7.3-fpm-lara.sock
listen.owner = lara    
listen.group = lara    
pm = dynamic
pm.max_children = 5
pm.start_servers = 2
pm.min_spare_servers = 1
pm.max_spare_servers = 3

Перезагружаем службу php-fpm

/etc/init.d/php7.3-fpm restart

Настройка ssl сертификата Lets Encrypt

Необходимо сформировать файл dhparam.pem. Процесс длится долго, до получаса на слабых виртуалках. Этот файл нужен для повышения безопасности и получения максимального рейтинга ssl.

openssl dhparam -out /etc/ssl/certs/dhparam.pem 4096

Скачиваем certbot-auto

cd ~
wget https://dl.eff.org/certbot-auto
mv certbot-auto /usr/local/bin/certbot-auto
chmod 0755 /usr/local/bin/certbot-auto

Первый запуск программы подтянит необходимые зависимости

certbot-auto --version

Список команд, доступных для certbot-auto

run             используется по умолчанию, если никакая команда не указана, получает и устанавливает сертификат;
certonly        только получает или обновляет сертификат, но не устанавливает его;
renew           обновляет сертификат;
enhance         добавляет настройки безопасности для существующих сертификатов;
certificates    отображает установленные сертификаты;
revoke          отзывает сертификат;
delete          удаляет сертификат;
register        создает ACME-аккаунт;

Выполняем команду получения сертификата

certbot-auto certonly

На первое сообщение выбираем 2. Если над этим сообщением есть ошибки, не обращаем на них внимание.
