Прием/обработка запросов пользователя, класс Request

Проучение объекта текущего запроса

Для получения объекта текущего запроса, используют внедрение зависимости.

Для этого в качестве параметра метода контролера (контроле как пример тут) необходимо передать объект Request с указанием его типа.

namespace App\Http\Controller\Admin;
use Illuminate\Http\Request;
use App\Http\Request;
use App\Http\Controllers\Controller;


class ContactController extends Controller
{
    public function show(Request $Request)
    {

    }
}

Когда Laravel будет создавать объект данного контроллера, из за включеной зависимости, он так же автоматически будет создавать объект запроса который будет передан в метод show.

Если в маршруте передаются параметры, в таком случаи принимаемы контролером параметры, должны указываться после указание объекта Request. Сам объект указывается первым параметром.

//Маршрут
Route::get('/contact/{id?}',['uses'=>Admin\ContactController@show]);

namespace App\Http\Controller\Admin;
use Illuminate\Http\Request;
use App\Http\Request;
use App\Http\Controllers\Controller;


class ContactController extends Controller
{
    public function show(Request $Request,$id)
    {

    }
}

Методы объекта Request

input - В качестве первого параметра, передаем название атрибута html, данные которого хотим получить. Во второй параметр, можно передать значение по умолчанию

$Request->input('name');

Для молучения значения конкретного атртбута, не обяхатеьно использовать метод input, можно обратиться к свойству класса Request соответствующий названию атрибута

$Request->name;

В свойства объекта Request попадают не только данные атрибутов но и параметры запроса. Если передатеся атрибут в запросе и параметр запроса и они имеют одинаковое название, в таком случаи выводится значение атрибута, но если значение атрибута пустое а параметр запроса присутствует, в таком случаи выводиться значение передаваемого параметра. (Видео)

Методы

has - Вернет True, если даный атрибут существует и он не пустой.

exists - Отличается от has, тем что не проверяет на пристуствие данных в передаваемом атрибуте.

all - Отображим все параметру, которые передаются в запросе в ввиде массива.

only - Указывает, какие параметры запроса необходимо вывести ввиде массива

except - Противоположность only, указаные параметры запроса не будут выводиться в массиве.

path - Возвращает URI запроса.

is - Поверяет запрос объекта, соотвествует ли он определенному шаблону (шаблон передается в качестве параметра метода)

url - Возвращает полный путь в адресной строке, без указания get параметров.

fullUrl - Возвращает полный путь вместе с писутствующими get параметрами.

method - Возвращает название типа запроса, которым были отправлены данные (get,post,pull,delete...)

isMethod - Возвращает истину, если типа запроса, соотвествует названию метода передаваемого первым параметром метода.

flash - Сохраняет все данные переданые в теле запроса в сессию. Видео

flush - Очищает данные в сессии.

old - Ранее сохраненные в сессию данные.

flashOnly - Перечисляет, какие атрибуты надо добавить в сессию.

flashExcept - Противоположный методу flashOnly.
